List 5 differences between Browser JS vs Node Js. 4. what happens when you type a URL in the address bar in the browser? Answer Difference between HTTP1.1 vs HTTP2 HTTP 1.1 was developed in 1989 by Tim Berners Lee and used till now. Work on HTTP 2 has started from 2015 only. Request and responses in HTTP/1.1 is transferred in plain-text messages where as in HTTP/2 it is first encoded into binary HTTP2 manages buffer space more properly and avoids buffer overflow While sending CSS & JS files HTTP 1.1 uses Resource inlining (which is not very useful for big files and has not way of stopping to send the CSS and JS files if the client already has them which causes burden on internet). Over here HTTP 2 uses “Server Push” as HTTP allows to cater multiple concurrent responses it can be selective as to which data to push (Which is better) Compression : HTTP 1.1 doesn’t compress Header which can be a burden in long time where as HTTP 2 uses HPACK compressor, it splits header from body and compresses both separately moreover HPACK can keep track of previously compressed files and can optimize it’s compression (Which is very good) HTTP version history HTTP was made by Tim Berners Lee in 1991(it had no version, but it is called as HTTP/0.9) HTTP/1.0 was made in 1996 HTTP/1.1 was made in 1997 (This is the version we still use today) HTTP/2.0 was made in 2015 HTTP/3.0 is in process Differences between Browser JS vs Node Js Browser JS is mainly used for client side application, but node.js can be used for both client and server side. Browser JS can run any engine but Node JS is run on V8 google chrome engine. Node JS can not add HTML tags Node JS is a runtime environment for Java Script language whereas Browser JS is language itself. Browser is subject to rigorous security policies and restrictions (browser) while the Node JS isn't. What happens when you type a URL in the address bar You enter URL in browser The Browser looks for IP address of domain name in DNS (Domain Name Server) Browser initiates a TCP connection Browser requests to server Server sends HTTP response Browser receives and displays website
